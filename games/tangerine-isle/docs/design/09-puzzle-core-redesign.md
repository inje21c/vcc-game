# 09 — 퍼즐 코어 재설계: 감귤섬 삼총사

작성일: 2026-07-12
상태: 구현 전 설계

## 1. 판단

현재 스테이지 흐름은 튜토리얼을 줄였지만, 퍼즐의 본질은 아직 약하다. 가장 큰 이유는 버튼 문이 한 번 열리면 계속 열리는 래치처럼 동작하기 때문이다. 이러면 돌을 버튼 위에 올리는 행동이 "상태를 유지하기 위한 희생"이 아니라 "한 번 수행하면 끝나는 체크리스트"가 된다.

이 게임은 소코반 계열의 압력, 캐릭터 능력 분업, 이동 체력, 폭탄 순서를 섞은 퍼즐이어야 한다. 따라서 버튼과 문은 다음 원칙을 가져야 한다.

> 돌이 버튼 위에 있을 때만 문이 열린다. 돌을 치우면 문은 다시 닫힌다.

이 한 문장이 게임을 바꾼다. 버튼은 열쇠가 아니라 비용이 되고, 돌은 도구가 아니라 묶여 있는 자원이 된다. 플레이어는 "이 돌을 계속 눌러둘 것인가, 다른 곳에 써야 하는가"를 고민해야 한다.

## 2. 정식 규칙 계약

### 2.1 압력 버튼

- 버튼은 오직 돌이 올라가 있을 때만 눌린다.
- 캐릭터가 버튼 위에 서 있는 것은 버튼 입력으로 보지 않는다.
- 귤, 열쇠, 폭탄, 상자는 버튼 입력으로 보지 않는다.
- 한 버튼 그룹 안의 모든 버튼이 눌려야 해당 그룹이 활성화된다.
- 버튼 그룹이 비활성화되면 그 그룹이 제어하는 문은 다시 닫힌다.

### 2.2 버튼 제어 문

- `D` 돌문: 버튼 그룹이 활성화된 동안만 열린다. 폭탄으로 파괴되지 않는다.
- `d` 나무문: 버튼 그룹이 활성화된 동안 열린다. 폭탄으로 파괴되면 영구 제거된다.
- 버튼으로 열린 `d`는 버튼이 풀리면 다시 닫힌다. 폭탄으로 파괴된 `d`만 영구 통로가 된다.
- 따라서 문 상태는 저장된 영구 플래그가 아니라 현재 돌 배치에서 파생되는 상태여야 한다.

### 2.3 문 닫힘의 안전 규칙

버튼 문은 즉시 닫히는 것이 기본이다. 단, 플레이어가 문 타일 위에 서 있는 순간 닫힘을 어떻게 처리할지는 반드시 명시해야 한다.

권장 규칙:

- 문 타일 위에 플레이어가 있어도 그룹이 비활성화되면 문은 닫힌 상태가 된다.
- 플레이어는 닫힌 문 위에 서 있을 수는 있지만, 그 칸을 떠난 뒤에는 다시 들어갈 수 없다.
- 돌이 열린 문 타일 위에 있을 때 문이 닫히는 상황은 레벨 데이터에서 금지한다.

이 규칙은 구현이 단순하고, 플레이어를 즉사시키거나 밀어내지 않는다. 다만 레벨 설계에서는 "문 위에 선 채 버튼을 해제해야 하는 해법"을 사용하지 않는다. 그런 해법은 논리적으로 지저분하고, 퍼즐의 쾌감보다 판정 의심을 만든다.

### 2.4 파생 상태와 영구 상태 구분

영구 상태:

- 먹은 귤
- 얻은 열쇠
- 열린 상자
- 밀려난 돌 위치
- 메워진 구덩이
- 터진 폭탄
- 폭탄으로 제거된 나무/나무문
- 토끼가 만든 땅굴

파생 상태:

- 버튼이 눌렸는지
- 버튼 제어 문이 현재 열렸는지

`openDoors`가 파생 상태라면, 솔버와 런타임은 매 이동 이후 같은 방식으로 재계산해야 한다. `openDoors`를 영구 상태처럼 저장하면 다시 "한 번 열면 끝" 문제가 재발한다.

## 3. 퍼즐의 본질

이 게임의 퍼즐은 "능력을 아는가"가 아니라 "자원을 어디에 묶어둘 것인가"다.

핵심 자원은 네 가지다.

- 돌: 버튼 유지, 구덩이 메우기, 이동 경로 차단/개방에 쓰이는 물리 자원
- 폭탄: 나무와 나무문을 영구 제거하는 1회성 자원
- 캐릭터 위치: 물, 땅굴, 돌 밀기 때문에 현재 누가 어디에 있는지가 제약이 됨
- 이동 체력: 정답은 가능하지만 우회가 길면 실패하게 만드는 압박

좋은 퍼즐은 이 네 가지 중 둘 이상을 동시에 묶는다. 예를 들어 "돌 하나는 문을 열어두는 데 묶여 있고, 다른 돌은 구덩이를 메워야 하며, 폭탄을 먼저 터뜨리면 돌을 버튼까지 가져갈 길이 사라진다" 같은 구조가 본 게임에 맞다.

## 4. 난이도 원칙

### 4.1 튜토리얼

`1-1` 하나만 튜토리얼이다.

목표:

- 고양이: 돌을 밀고 구덩이를 메운다.
- 버튼: 돌을 올려둔 동안만 문이 열린다는 사실을 보여준다.
- 폭탄: 나무문을 영구 제거한다.
- 거북이: 물 안의 열쇠를 가져온다.
- 토끼: 한 칸 나무벽을 통과한다.
- 열쇠와 상자: 스테이지 클리어 루프를 닫는다.

튜토리얼의 목적은 설명이 아니라 문법 습득이다. 이후 스테이지에서는 같은 규칙을 말로 반복하지 않는다.

### 4.2 본 퍼즐

`2-1`부터는 다음 조건 중 최소 3개를 포함해야 한다.

- 버튼 문이 닫히는 것을 이용한 돌 고정 비용
- 구덩이에 넣으면 사라지는 돌과 버튼에 묶어야 하는 돌 사이의 선택
- 폭탄 순서 오류로 생기는 데드락
- 물 위 전환 잠금으로 인한 캐릭터 순서 제약
- 토끼 땅굴 위치가 이후 이동 동선에 영향을 주는 구조
- 열쇠 방과 상자 방이 단순 직선이 아닌 회귀 동선을 요구

## 5. 스테이지 구조 제안

### 5.1 2-1 — 귤밭 창고: 첫 진짜 압력 퍼즐

목표 감정:

처음에는 쉬워 보인다. 돌 두 개와 버튼 두 개가 보인다. 하지만 하나를 잘못 쓰면 문은 다시 닫히고, 나중에 필요한 돌이 부족해진다.

구조:

```
[A 창고 입구] --버튼문--> [B 토끼 복도]
      |                      |
      |                      v
[C 물 저장고] <---------- [D 상자방]
```

필수 설계:

- A에는 돌 3개, 버튼 2개, 구덩이 1개가 있다.
- 버튼 2개는 B로 가는 돌문을 연다.
- 돌 2개를 버튼에 묶어야 하므로 남는 돌 1개만 구덩이에 쓸 수 있다.
- 구덩이를 잘못 메우면 열쇠 회수 후 돌아오는 동선이 길어져 체력이 빠듯해진다.
- B는 토끼가 통과해야 하지만, 토끼로 들어간 후 상자방에 바로 갈 수는 없다.
- C의 물 저장고는 거북이로만 열쇠를 얻는다.
- D의 상자는 열쇠가 있어야 열리며, 열쇠 없이 먼저 도달하면 "아, 경로가 아니라 순서가 문제구나"를 느끼게 한다.

정답 논리:

1. A에서 돌 두 개를 버튼에 올려 B 문을 연다.
2. 남은 돌 하나로 구덩이를 메워 C 왕복 동선을 줄인다.
3. B에서 토끼로 나무벽을 통과해 D를 확인한다.
4. D에서 상자를 보고 열쇠가 필요함을 확인한다.
5. C로 돌아가 거북이로 열쇠를 회수한다.
6. 다시 D로 가서 클리어한다.

금지:

- 버튼 두 개를 누른 뒤 돌을 치워도 B 문이 계속 열린다면 이 스테이지는 실패다.
- C에서 바로 D로 가는 우회가 있으면 압력 퍼즐이 무너진다.

### 5.2 3-1 — 화산 귤길: 폭탄과 압력문의 결합

목표 감정:

"문을 여는 법"은 알지만, 이번에는 어떤 문을 일시적으로 열고 어떤 문을 영구 파괴할지 선택해야 한다.

구조:

```
[A 폭탄 전실] --나무문/폭탄--> [B 내부 압력실] --버튼문--> [C 물 열쇠+상자]
```

필수 설계:

- A에는 폭탄 2개가 있다.
- 첫 폭탄은 A-B 나무문을 여는 데 필요하다.
- 두 번째 폭탄은 B 안쪽의 나무 장애물을 제거해야 한다.
- 폭탄을 잘못된 순서로 터뜨리면 버튼으로 돌을 보내는 길이 막힌다.
- B에는 토끼가 먼저 통과해야 보는 영역과, 고양이가 나중에 돌을 밀어야 하는 영역이 분리되어 있다.
- B의 버튼문은 C 입구를 제어한다. 돌이 버튼에서 벗어나면 C 입구는 닫힌다.
- C에는 물 한가운데 열쇠, 같은 방 안쪽에 상자가 있다.

정답 논리:

1. A에서 첫 폭탄으로 나무문을 제거한다.
2. 토끼로 B의 한 칸 나무벽을 넘어 내부 구조를 연다.
3. 고양이로 돌을 버튼까지 밀어 C 입구를 계속 열어둔다.
4. 거북이로 C 물길을 건너 열쇠를 얻는다.
5. 같은 방 상자로 이동해 클리어한다.

금지:

- C 입구 문이 한 번 열린 뒤 계속 열리면 퍼즐의 핵심이 사라진다.
- 폭탄이 단순 통행권이면 안 된다. 폭탄은 "순서가 틀리면 망한다"는 역할을 가져야 한다.

### 5.3 4-1 — 두 개의 저울

목표 감정:

처음으로 "둘 다 열어둘 수 없다"는 선택을 강제한다.

구조:

```
[A 중앙 허브]
  ├─ [B 왼쪽 저울: 열쇠 경로]
  ├─ [C 오른쪽 저울: 상자 경로]
  └─ [D 돌 재배치 방]
```

필수 설계:

- 버튼 그룹 `left_gate`와 `right_gate`가 있다.
- 사용 가능한 돌 수가 두 그룹을 동시에 유지하기에 부족하다.
- 먼저 열쇠 경로를 열고, 이후 돌 배치를 바꿔 상자 경로를 열어야 한다.
- 단, 돌을 치우는 순간 이전 문이 닫히므로 플레이어가 어느 쪽에 갇히지 않도록 돌아오는 동선을 미리 확보해야 한다.
- 토끼 땅굴은 돌 재배치 방으로 돌아오는 숏컷이다.

정답 논리:

1. 왼쪽 문을 열어 열쇠를 얻는다.
2. 토끼 숏컷으로 중앙으로 돌아온다.
3. 돌을 오른쪽 버튼 그룹으로 재배치한다.
4. 오른쪽 문을 열어 상자로 간다.

여기서부터 퍼즐은 "도착"보다 "되돌아올 수 있는가"가 핵심이 된다.

## 6. 솔버와 검증 설계

버튼 문이 닫히면 솔버의 역할이 더 중요해진다.

필수 검증:

- 버튼 그룹 활성/비활성에 따라 문 통행 가능성이 매 상태에서 재계산된다.
- 열렸던 문을 닫는 행동 이후에도 해법이 남는지 확인한다.
- 돌이 버튼에서 빠진 뒤 문이 닫히는 경로를 공식 해법에 포함한다.
- `solutionPath`는 단순 클리어 경로뿐 아니라 핵심 퍼즐 상태 변화를 지나야 한다.
- 우회 최단해가 퍼즐을 생략하면 실패로 본다.

추가 출력:

- 필수 버튼 유지 시간: 몇 스텝 동안 어떤 버튼이 눌려 있어야 하는지
- 버튼 해제 횟수: 해법 중 문이 다시 닫히는 순간 수
- 데드락 후보: 버튼에서 돌을 빼거나 폭탄을 터뜨린 뒤 해법이 사라지는 상태 수

## 7. 구현 순서 제안

아직 구현하지 않는다. 다음 순서로 진행한다.

1. GDD에 "버튼은 압력식이며 돌을 치우면 문이 닫힌다"를 정식 반영한다.
2. 레벨 포맷 문서에 `openDoors`가 파생 상태임을 반영한다.
3. 솔버 상태 정의에서 버튼 문 개방 상태를 저장하지 않고 돌 위치에서 계산하도록 설계한다.
4. 런타임 규칙 엔진의 버튼 재평가 정책을 구현한다.
5. 기존 1-1을 단일 튜토리얼로 검증한다.
6. 2-1을 첫 본 퍼즐로 재설계한다.
7. 3-1을 폭탄+압력문 조합 퍼즐로 재설계한다.
8. 각 스테이지에서 "우회 최단해"가 없는지 솔버로 확인한다.

## 8. 설계 합격 기준

스테이지는 다음 질문에 답할 수 있어야 합격이다.

- 이 스테이지에서 플레이어가 고민해야 하는 선택은 무엇인가?
- 어떤 자원이 묶여서 다른 행동을 못 하게 만드는가?
- 실수했을 때 왜 막혔는지 눈으로 이해되는가?
- 정답이 단순 직선 이동이 아니라 상태 재배치인가?
- 버튼 문이 다시 닫히는 순간이 퍼즐 의미를 만드는가?
- 열쇠와 상자가 단순 수집물이 아니라 동선 설계를 요구하는가?

이 질문에 답하지 못하는 스테이지는 튜토리얼이거나 산책로다. 본 게임의 스테이지로 보지 않는다.

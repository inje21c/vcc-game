## 1. 최종 방향

제주삼다수 브랜드 게임은 별도 신규 게임이 아니라 `games/tangerine-isle`의 검증된 퍼즐 엔진을 활용한 **브랜드 에디션**으로 개발한다.

핵심 정의:

> 읽던 제주삼다수 스토리를, 직접 걷고 밀고 풀어보는 플레이어블 브랜드 경험으로 전환한다.

따라서 목표는 “삼다수 로고를 붙인 미니게임”이 아니다. 기존 감귤섬 삼총사의 규칙 어휘를 제주삼다수의 지층, 물길, 청정함, 보전 서사로 번역하는 것이다.

## 2. 제품 포지션

가칭은 **제주삼다수: 한 방울의 시간**으로 둔다.

핵심 카피:

> 180만 년의 땅, 31년의 물길

사용자는 수원지를 구하는 영웅이 아니라 **한라산 고지에 내린 빗방울 하나의 31년 여정을 안내하는 동행자**다. 위기, 오염, 고갈, 정화 서사는 사용하지 않는다.

## 3. 반드시 지킬 원칙

1. **엔진은 최대한 바꾸지 않는다.**  
   `rules.js`, 레벨 포맷, 솔버의 계약을 유지하고 표시명, 에셋, 카피, 지식 카드 데이터로 브랜드 의미를 입힌다.

2. **물은 소모품이 아니다.**  
   물방울은 체력이나 화폐가 아니라 보호하고 안내할 대상이다. 기존 `hp`는 대외 표시에서 `탐사력`으로 바꾸고, 회복 아이템은 감귤로 유지한다.

3. **브랜드 정보는 행동 뒤에 온다.**  
   긴 설명을 먼저 보여주지 않는다. 퍼즐 행동, 결과, 짧은 팩트, 선택형 상세 카드 순서로 전달한다.

4. **구매와 완주는 분리한다.**  
   구매 인증, 쿠폰, 공병 회수는 보너스다. 비구매자도 본편 서사를 완주할 수 있어야 한다.

5. **브랜드 리스크 표현을 금지한다.**  
   오염된 물, 마른 수원, 폭발하는 한라산, 검은 폐수, 제품 병을 무기나 폭탄으로 쓰는 표현은 사용하지 않는다.

## 4. 기존 엔진의 브랜드 번역

| 기존 감귤섬 요소 | 제주삼다수 브랜드 에디션 |
|---|---|
| 체력 `hp` | 탐사력 |
| 감귤 | 탐사력 회복 아이템 |
| 돌 밀기 | 화산석·지층 블록 배치 |
| 폭탄 | 지층 공명석 |
| 폭발 | 파동 활성화 |
| 물 타일 | 암석 틈의 물길 |
| 토끼 땅굴 | 스며드는 틈 탐색 |
| 버튼 | 저류 노드·관측 노드 |
| 돌문 | 암반 경계 |
| 열쇠 | 물길 문양 |
| 상자 | 지질·청정 기록 아카이브 |

내부 ID는 유지하되 화면 표시와 에셋, 지식 카드, 힌트 문구를 바꾼다. 이 방향이 파일럿 리드타임을 가장 짧게 만든다.

## 5. 챕터 구조

브랜드 스토리는 5장 구조로 설계한다.

| 장 | 브랜드 의미 | 플레이 동사 | 대표 구현 |
|---|---|---|---|
| 프롤로그: 제주도 형성 | 물을 품을 수 있는 섬과 지층 | 쌓다, 맞추다, 발견하다 | 화산석 배치, 지층 결합점 |
| 1장: 스며듦 | 빗물이 제주 지질 틈으로 들어감 | 찾다, 통과하다, 연결하다 | 토끼 통로, 투수 통로 |
| 2장: 머금음 | 암석 틈과 저류 공간을 통한 느린 이동 | 머물다, 균형 맞추다 | 물길, 저류 노드, 돌 배치 |
| 3장: 청정함 | 자연 조건과 품질 확인의 신뢰 | 확인하다, 연결하다 | 관측 노드, 버튼 그룹 |
| 4장: 소중함 | 연구, 관리, 보호, 순환 참여 | 관찰하다, 지키다, 함께하다 | 관측 퍼즐, 공동 보호 지도 |

코어 본편은 8~10스테이지로 제한한다. 31년은 스테이지 수가 아니라 31일 라이브 챌린지의 운영 상징으로 사용한다.

## 6. MVP 범위

첫 개발 목표는 완성 캠페인이 아니라 **제안용 모바일 웹 데모**다.

포함:

- 단면형 타이틀 화면
- 스며듦 대표 스테이지 1개
- 머금음 대표 스테이지 1개
- 소중함 대표 스테이지 1개
- 지식 카드 6종
- 엔딩 단면 애니메이션
- 로그인 없는 플레이
- 기본 행동 이벤트 설계

제외:

- 실시간 지하수 데이터
- 대규모 리더보드
- 주문, 결제, 개인정보 연동
- 온팩 QR 대량 운영
- 31일 전체 콘텐츠
- 공식 브랜드 캐릭터 전면 재개발

## 7. 구현 순서

### 1단계. 브랜드 설정 데이터 분리

`tangerine-isle`에 브랜드 에디션 설정을 추가한다.

- 표시명 사전: `hp` → `탐사력`, `bomb` → `지층 공명석`
- 지식 카드 JSON
- 챕터별 힌트 문구
- 승인 상태와 출처 메타데이터

공식 숫자와 카피는 코드에 직접 박지 않는다.

### 2단계. 에셋 스킨 레이어 추가

규칙 ID는 그대로 두고 렌더러가 에디션별 에셋 맵을 참조하게 한다.

- 기본 에디션: 감귤섬 삼총사
- 브랜드 에디션: 제주삼다수

이 구조가 되면 향후 다른 브랜드 제안에도 엔진을 재사용할 수 있다.

### 3단계. 대표 스테이지 3개 제작

기존 `stages.json` 구조를 유지하면서 새 파일럿 스테이지를 별도 ID로 만든다.

- `samdasoo-1`: 스며듦
- `samdasoo-2`: 머금음
- `samdasoo-3`: 소중함

각 스테이지는 솔버 검증을 통과해야 하며, 하나의 핵심 통찰만 갖는다.

### 4단계. 지식 카드와 엔딩 연결

스테이지 클리어 뒤 2문장 이하의 팩트 카드를 보여준다. 상세 설명은 선택으로 둔다.

엔딩은 사용자가 지나온 지층 단면을 연결하고, 마지막에 제품을 여정의 결과로 절제되게 노출한다.

### 5단계. 분석 이벤트 추가

파일럿에서 필요한 이벤트만 최소로 넣는다.

- 게임 진입
- 스테이지 시작
- 스테이지 클리어
- 재시작
- 지식 카드 열람
- 엔딩 도달
- 앱/외부 CTA 클릭

## 8. 성공 기준

파일럿은 절대 수치 약속보다 기준선 대비 상대 향상으로 판단한다.

검증할 가설:

1. 일반 랜딩이나 룰렛보다 체류 시간이 길다.
2. 핵심 브랜드 팩트 회상이 높다.
3. 사용자가 다음 콘텐츠나 앱 이동을 선택한다.
4. 레벨 데이터 중심으로 운영 콘텐츠를 빠르게 추가할 수 있다.

## 9. 의사결정 기준

새 기능을 넣을 때는 아래 질문을 통과해야 한다.

1. 이 기능이 제주삼다수의 브랜드 진실을 플레이 규칙으로 번역하는가?
2. 엔진 계약을 깨지 않고 데이터와 표시 레이어로 처리 가능한가?
3. 사용자가 구매하지 않아도 본편을 완주할 수 있는가?
4. 공식 사실과 창작 은유가 분리되어 있는가?
5. 1인 운영자가 캠페인 후에도 유지할 수 있는 규모인가?

하나라도 명확히 답하지 못하면 MVP 이후로 미룬다.

## 10. 다음 작업

1. `tangerine-isle`에 브랜드 에디션 설정 파일 구조를 설계한다.
2. 에셋 맵과 표시명 사전을 추가한다.
3. 스며듦, 머금음, 소중함 대표 레벨을 솔버 기준으로 설계한다.
4. 지식 카드 6종을 JSON으로 작성한다.
5. 제안 데모용 타이틀과 엔딩 흐름을 만든다.

이 방향의 핵심은 단순하다. 먼저 기존 퍼즐 엔진을 깨지 않고 제주삼다수의 의미가 플레이 안에서 자연스럽게 느껴지는 3스테이지 데모를 만든다. 그 다음에 브랜드 승인, 앱 연동, 31일 챌린지, 온팩 QR로 확장한다.

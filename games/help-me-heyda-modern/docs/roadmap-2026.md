# 도와줘 헤이다 2026 진행 계획

> **방향 전환 (2026-06):** 챕터 보스 확장·아이템 시스템은 v2.0으로 이월한다. v1.0은 스테이지 36개 + 서바이벌 전환으로 클로징한다.

## 현재 기준

`도와줘 헤이다 2026`은 원작 자료를 바탕으로 현대 모바일 브라우저에서 플레이 가능한 형태까지 구축되어 있다. 기본 메뉴, 스토리 화면, 스테이지 플레이, 서바이벌, 결과 화면, Web Audio 기반 사운드, PWA manifest와 service worker가 들어가 있다.

최근 안정화 기준:

- JavaScript 문법 검사 통과
- 브라우저 전역 기준 정적 검사 통과
- public 에셋 참조 누락 없음
- service worker 캐시 목록 누락 없음
- 주요 액션 런타임 스모크 테스트 통과
- stage 파일 로딩 실패 시 fallback stage로 복구
- 타이틀, 메인메뉴, 스토리모드 1~18 스테이지, 결과/엔딩 흐름 1차 완성
- 스토리모드는 3개 챕터 단위 진행 구조로 전환 중

## v1.0 개발 우선순위

1. 스테이지 19–36 절차적 생성 구현 ← **현재 집중**
2. 난이도 커브 연장 및 플레이 검증
3. Stage 36 클리어 → 서바이벌 전환 유도 UI
4. 전체 클로징 체크리스트 통과

## v1.0 난이도 설계 기준

목표:

- 처음 하는 사람은 Stage 1-6에서 조작과 3개 매칭을 익힌다.
- Stage 7-18은 4개, 5개, 6개 매칭으로 넘어가며 바닥 슬롯 관리와 콤보 유지가 핵심이 된다.
- Stage 19-24는 새 챕터 적응 구간이다. 6개 매칭은 유지하되 셔플 없이 48-54블록, 4-5종으로 다시 리듬을 잡는다.
- Stage 25-30부터 예고형 셔플을 도입한다. 3초 전 경고를 주고 1-2쌍만 섞어 계획 수정 능력을 시험한다.
- Stage 31-36은 고수 구간이다. 66-72블록, 6-7종, 짧아지는 예고형 셔플로 빠른 판단과 회복력을 요구한다.

현재 적용:

- 정화 스킬과 관련 보상 표시는 v2.0으로 숨긴다.
- Stage 19 이후 절차 생성은 모든 타입이 6의 배수로 배치되어 남는 블록 없이 클리어 가능해야 한다.
- 난이도 상승은 단순히 시간을 줄이는 방식이 아니라 `매칭 수 → 보드 밀도 → 팔레트 → 예고형 셔플` 순서로 체감되게 한다.
- Stage 18처럼 보드가 가득 찬 고난도 구간은 정답 수행 퍼즐이 되지 않도록 전용 마스터리 보드로 재구성한다. 단, 게임은 왼쪽 정렬 규칙이 핵심이므로 앞칸이나 중간칸을 비우지 않는다. 현재 Stage 18은 Stage 19의 읽을 수 있는 배치감을 앞당겨 60블록, 5종, 각 문양 12개로 구성하고, Chapter 3 이후 상단이 막힌 상황의 첫 불일치는 즉사 대신 바닥 비우기와 시간 패널티로 처리한다.

## 1단계. 오류 안정화

목표:

- 게임 시작, 메뉴 이동, 스테이지 진입, push, 클리어, 게임오버가 예외 없이 동작해야 한다.
- 모바일 화면에서 UI가 겹치지 않아야 한다.
- 스테이지 데이터가 없거나 fetch에 실패해도 게임이 멈추지 않아야 한다.

체크리스트:

- `node --check games/help-me-heyda-modern/src/game2026.js`
- 브라우저 기준 ESLint `no-undef`
- public 에셋 누락 검사
- service worker 캐시 에셋 누락 검사
- 스토리/서바이벌 액션 스모크 테스트
- 실제 모바일 브라우저 수동 테스트

## 2단계. 스토리모드 챕터 분리

목표:

- 18개 스테이지를 실제 챕터 진행 구조로 확장한다.
- 각 챕터는 6개 스테이지와 보스 후보 스테이지를 가진다.
- 챕터 클리어 후 다음 챕터 컷신과 보상이 열린다.

1차 구현 구조:

- Chapter 1: 숲이 숨을 멈춘 날, Stage 1-6, 보스 후보 Stage 6
- Chapter 2: 토템의 노래, Stage 7-12, 보스 후보 Stage 12
- Final Chapter: 마지막 길, Stage 13-18, 보스 후보 Stage 18

현재 적용:

- 처음 시작 또는 처음부터 시작 시 Chapter 1 인트로를 보여준다.
- Stage 6 클리어 후 Chapter 2 인트로를 보여준다.
- Stage 12 클리어 후 Final Chapter 인트로를 보여준다.
- 저장된 진행도 기준으로 열린 챕터까지만 Prev/Next 탐색이 가능하다.
- 챕터 데이터에 `bossStage`와 `boss` 필드를 추가해 다음 단계 보스 작업의 기준으로 삼는다.

다음 작업:

- 챕터 클리어 보상 연결
- 보스 스테이지 전용 규칙 추가
- 챕터별 배경/BGM/효과음 분리
- Chapter 1 클리어 보상은 `자연의 힘: 잎의 정화`로 설계한다.

## 3단계. 챕터 보스

목표:

- 보스는 단순 HP가 아니라 퍼즐 규칙을 흔드는 존재여야 한다.
- 챕터마다 다른 압박 패턴을 가진다.

보스 후보:

- 측량 기사: 특정 줄을 잠근다.
- 벌목 장비: 일정 시간마다 블록을 위로 민다.
- 불도저: 시간 압박과 바닥 라인 방해를 강화한다.
- 개발 책임자: 여러 패턴을 섞는 최종 보스.

Chapter 1 보스 1차 설계:

- 설계 문서: `games/help-me-heyda-modern/docs/chapter1-boss-design.md`
- Stage 6 보스: 붉은 깃발의 감독관
- 핵심 패턴: 특정 줄에 붉은 깃발 경고를 띄우고, 무시하면 시간 패널티를 준다.
- 첫 구현은 이미지/BGM 없이도 Canvas 도형과 Web Audio로 동작해야 한다.

## 4단계. 토템 아이템과 스킬

목표:

- 토템 문양마다 플레이 감각이 달라지게 한다.
- 스킬은 자동 승리가 아니라 위기 대응과 콤보 설계 도구여야 한다.

초기 스킬 후보:

- 잎 토템: 한 줄의 첫 블록을 정화한다.
- 해 토템: 바닥 라인의 같은 문양을 강화한다.
- 물결 토템: 한 줄을 한 칸 아래로 안정화한다.
- 산 토템: 상승 압박을 한 번 막는다.
- 불 토템: 선택한 블록 하나를 제거한다.
- 구름 토템: 다음 블록 미리보기.

Chapter 1 보상 설계:

- 설계 문서: `games/help-me-heyda-modern/docs/chapter1-reward-nature-power.md`
- Stage 6 보스 클리어 후 `자연의 힘: 잎의 정화`를 해금한다.
- Chapter 2부터 `개발의 흔적` 방해물이 등장하고, 잎의 정화로 제거한다.

## 5단계. BGM과 효과음

목표:

- 현재 절차적 Web Audio 구조를 유지하되 모드별 레이어를 분리한다.
- 추후 실제 음원으로 교체하기 쉬운 인터페이스를 둔다.

필요 사운드:

- 타이틀
- 메뉴 선택
- 스토리 컷신
- 스테이지 시작
- push
- 매치 성공
- 콤보 상승
- 실패
- 보스 등장
- 클리어
- 게임오버

## 6단계. 서바이벌 모드 개선

목표:

- 단순 시간 경과 난이도보다 플레이 실력에 반응하는 난이도로 개선한다.
- 오래 버틸수록 새로운 토템과 압박 패턴이 열린다.
- 초반에 보드를 너무 빨리 정리한 플레이어가 기다리지 않도록 `Rush` 버튼으로 즉시 라인을 당겨오고 보너스 점수를 받을 수 있게 한다.

개선 후보:

- 콤보가 높으면 보상 아이템 등장
- 실수가 많으면 상승 속도 증가
- 일정 점수마다 보스 압박 이벤트
- 랭킹 또는 로컬 최고 기록
- 1분, 3분, 5분 생존 보상

## 7단계. 미션 모드

목표:

- 짧은 목표를 가진 반복 플레이 모드로 만든다.
- 베타 테스트 때 밸런스 확인에 활용한다.

미션 후보:

- 제한 시간 안에 특정 문양 3회 클리어
- 실수 없이 5콤보 달성
- 특정 스테이지를 남은 시간 300 이상으로 클리어
- 보스 방해를 받고도 클리어
- 스킬을 1회 이하로 사용해 클리어

## 베타 테스트 운영

- 안정 테스트는 Production URL에서 받고, 새 기능 검증은 Preview URL에서 먼저 받는다.
- P0/P1 오류는 새 기능보다 먼저 처리한다.
- 수정 후 검증 명령과 수동 테스트 결과를 리포트에 남긴다.

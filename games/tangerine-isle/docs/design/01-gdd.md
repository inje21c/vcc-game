## 1. 게임 개요

| 항목 | 내용 |
|---|---|
| 장르 | 탑다운 타일 기반 퍼즐 (리얼타임 조작, 자원은 결정론적) |
| 화면 | 룸(room) 단위 고정 화면. 하나의 스테이지 = 여러 룸으로 쪼개진 큰 맵 |
| 플랫폼 | 모던 웹 / PWA (Canvas 2D, vanilla JS) |
| 규모 | 4챕터 × 4스테이지 + 최종 화산 스테이지 = 17 스테이지 (간단하게 시작, 확장 가능 구조) |
| 클리어 단위 | 스테이지당 열쇠 1개 확보 → 보물상자 1개 개방 |
| 전체 목표 | 모든 보물상자를 열어 화산 폭발(저주)을 막는다 |

## 2. 서사

### 2.1 시놉시스

감귤섬 중앙의 화산은 오랫동안 잠들어 있던 휴화산이었다. 바다 건너의 악당 세력이 자신들의 바다 영역을 넓히기 위해 섬을 가라앉히려 저주를 걸었다 — 섬의 혈자리마다 저주의 강철 말뚝을 박아 섬의 정기를 끊고 화산을 깨우려 한다. 저주를 풀 수 있는 힘은 섬 곳곳에 숨겨진 보물상자 안에 봉인되어 있다. 흰 고양이, 갈색 토끼, 초록 거북이 — 감귤섬 삼총사만이 모든 보물을 되찾아 화산의 분노를 잠재울 수 있다.

### 2.2 표현 수위 (확정)

역사적 모티프(쇠말뚝 설화, 한라산)는 서사의 뿌리로 유지하되 게임 내 표기는 우화로 완결한다: 악당은 "바다 건너 강철 세력", 화산은 "섬의 큰 산". 저주 말뚝(`o_stake`)은 순수 서사 장치로, 게임플레이 기능과 클리어 연출 없이 챕터 배경·스토리 컷에만 등장한다 (Q7 확정).

### 2.3 서사-메커니즘 동기화

- 보물상자 = 저주 해제의 봉인 → 전량 수집이 곧 진행도
- 스테이지 클리어 누적 → 화산 연기가 잦아드는 월드맵 연출
- 최종 스테이지: 화산 내부, 마지막 봉인

## 3. 캐릭터 — 능력 매트릭스 (확정)

| 표시명 (확정) | **라봉이** 🐱 | **금향이** 🐰 | **청귤이** 🐢 |
|---|---|---|---|
| 명명 근거 (감귤 품종 시스템) | 한라봉 — 화산(섬의 큰 산) 서사와 직결, 리더 | 황금향 — 갈색 털, 땅굴 | 청귤 — 초록 등껍질, 물(靑) |
| 내부 ID (불변) | `cat` | `rabbit` | `turtle` |

**명명 원칙**: 표시명은 데이터(교체 가능), 내부 ID는 계약(불변). 코드·레벨 데이터·에셋 파일명은 전부 `cat`/`rabbit`/`turtle`을 쓰고, 표시명은 로컬라이즈 테이블 한 곳에서만 관리한다.

| 능력 | 라봉이 🐱 | 금향이 🐰 | 청귤이 🐢 |
|---|---|---|---|
| 돌(토템) 밀기 | **O** | X | X |
| 나무 1칸 땅굴 관통 | X | **O** | X |
| 수영 (물 이동) | X | X | **O** |
| 이동 / 귤 먹기 / 열쇠 획득 / 상자 열기 / 폭탄 기폭 | O | O | O |

- **시작 캐릭터는 항상 고양이** (Q2 확정).
- 토끼의 땅굴 규칙 (Q4 확정): **나무 벽 앞에서만** 발동. 두께 1칸의 나무를 사이에 두고 파고 들어가 반대편으로 나온다. 역방향 재통과 가능. 즉 "땅굴"과 "나무 통과"는 하나의 능력이며, 두께 2칸 이상의 나무 벽은 관통 불가.

## 4. 귤 경제 — 이동 체력 시스템 (Q2 확정, 본 게임의 핵심 자원 구조)

원작의 "귤 = 캐릭터 전환 비용"을 폐기하고, **귤 = 이동 체력**으로 재설계한다.

| 규칙 | 값 |
|---|---|
| 체력의 의미 | 남은 이동 가능 칸수 |
| 소모 | 1칸 이동 = 체력 1 (룸 간 이동 포함) |
| 회복 | 귤 1개 획득 = 체력 +G (기본값 G=10, 레벨 튜닝 변수) |
| 시작 체력 | 스테이지별 지정 S₀ — 초반 스테이지는 넉넉하게 |
| 캐릭터 전환 | **무제한·무비용.** 언제든 현재 위치에서 교대 호출. 활성 캐릭터는 항상 1명 |
| 게임오버 | 체력 0 도달 → 스테이지 재시작 |

### 4.1 구조적 함의 — 난이도가 단일 변수로 환원된다

이동이 곧 자원이므로 게임의 경제는 완전히 결정론적이다. 레벨 검증기(솔버)가 최단 해법 스텝 수 `MinSteps`를 계산하면, 스테이지 난이도는 여유율 하나로 정의된다:

```
여유율 α = (S₀ + 귤총량 × G) / MinSteps
```

제안 기준선: 튜토리얼 α ≥ 2.0 / 중반 1.5 / 후반 1.2~1.3. 레벨 설계(03)에서 스테이지마다 α를 명시하고, 솔버가 자동 산출한 α와 대조하는 것을 설계 완료 조건으로 삼는다.

**파생 요구사항 — 격자 가독성 (2026-07-09 추가)**: 이동이 곧 자원이므로 플레이어는 화면에서 칸 수를 세며 동선을 계획한다. 따라서 타일 경계가 시각적으로 명확해야 하는 것은 미학이 아니라 메커니즘 요구사항이다. 지형 타일은 테두리가 보이는 A버전을 정본으로 하고(물만 연속면 예외), 렌더러는 격자 가독성을 해치는 효과(과도한 블렌딩·안개 등)를 금지한다.

### 4.2 타이머 폐지

이동 체력이 실패 조건과 압박 장치를 겸하므로 **시간 제한은 두지 않는다** (이전 버전 §9 대체). 실시간 타이머와 스텝 예산이 공존하면 압박 장치가 이중화되어 퍼즐의 "생각할 자유"를 해친다. 점수는 `잔여 체력`으로 산정한다 — 효율적 동선이 곧 고득점.

## 5. 타일·오브젝트 명세 (확정)

### 5.1 지형

| ID | 이름 | 규칙 |
|---|---|---|
| `t_ground` | 바닥 | 전 캐릭터 이동 가능 |
| `t_tree` | 나무 | 차단. 토끼만 두께 1칸 땅굴 관통. 폭탄으로 제거 가능 |
| `t_water` | 물 | 차단. 거북이만 수영 이동. 돌을 밀어 넣어도 다리가 되지 않음 |
| `t_pit` | 구덩이 | 차단. 돌을 밀어 넣으면 메워져 전 캐릭터 통행 가능 |
| `t_door_wood` | 나무 비밀문 | 차단. 개방: 연결된 버튼 그룹 완성 **또는** 폭탄 파괴 (Q5) |
| `t_door_stone` | 돌 비밀문 | 차단. 개방: 연결된 버튼 그룹 완성만 가능. 폭탄 무효 (Q5) |

### 5.2 오브젝트

| ID | 이름 | 규칙 |
|---|---|---|
| `o_rock` | 돌 / 토템 | 고양이만 1칸씩 밀기(당기기 불가). 용도: 버튼 누르기, 구덩이 메우기. 폭발에 파괴되지 않음 (Q9) |
| `o_button` | 퍼즐 버튼 | 그룹 단위. 그룹 전체에 돌이 올라가야 연결된 비밀문 개방 |
| `o_bomb_N` | 폭탄 | §6. N = 십자 4방향 각 N타일 폭발 범위 |
| `o_tangerine` | 귤 | 획득 = 체력 +G. 폭발 무시 (Q9) |
| `o_key` | 열쇠 | 획득 = 상자 개방 자격. 폭발 무시 (Q9) |
| `o_chest` | 보물상자 | 열쇠 보유 상태로 접촉 시 개방 = 스테이지 클리어 |
| `o_stake` | 저주 말뚝 | 순수 서사 장치. 게임플레이 기능 없음 (Q7) |

## 6. 폭탄 연쇄 퍼즐 (확정)

1. **기폭 = 플레이어 접촉** (Q6): 캐릭터가 폭탄 칸에 진입하면 즉시 폭발.
2. 기폭 방식의 논리적 귀결로, **폭발은 플레이어를 해치지 않는다** — 플레이어는 항상 폭심에 있으므로 폭발 피해가 존재하면 모든 기폭이 자살이 된다. 실패 조건은 체력(§4)과 데드락(§6-6)만이다.
3. 폭탄 N은 4방향(십자)으로 각 N타일 범위를 폭발시킨다. 범위 내 나무·나무 비밀문은 제거된다. 돌 비밀문·돌·귤·열쇠·상자는 영향 없다.
4. 범위 내의 다른 폭탄은 즉시 연쇄 기폭된다 (자체 범위로 재폭발).
5. 폭탄은 1회성 소모 자원이다. 연쇄로 조기 소모되면 되돌릴 수 없다.
6. 퍼즐의 본질 = **기폭 순서 설계**. 각 폭탄에는 맡아야 할 파괴 대상이 있고, 잘못된 연쇄로 필요 대상이 남은 채 폭탄이 소진되면 데드락이다. 런타임은 데드락을 감지해 재시작을 안내한다 (감지 엔진 = 레벨 검증기와 동일 코어).

## 7. 맵 구조

- 스테이지 = 큰 맵 = 룸(15×15 타일) 격자 분할. 룸 가장자리 이동으로 인접 룸 전환. 룸 전환도 이동 1칸 = 체력 1.
- 스테이지 내 상태 지속: 밀어둔 돌, 터뜨린 폭탄, 연 문, 먹은 귤은 룸을 나갔다 와도 유지.
- 열쇠와 상자는 서로 다른 룸에 배치해 탐색 동선을 형성.
- 룸 수 성장: 챕터1 = 2×1~2×2 → 챕터4 = 3×3. 상세는 03 문서.

## 8. 미니게임 — 열기구 횡스크롤

스테이지 간 장거리 이동 시점의 막간. 폭탄 회피 + 귤 획득, 실패해도 진행이 막히지 않는 보상형 (획득 귤 = 다음 스테이지 시작 체력 보너스 +G×획득수). 구현은 P4 후반, 스테이지 전환 인터페이스만 선반영.

## 9. 실패·재시작 (v1.0에서 재정의)

| 조건 | 처리 |
|---|---|
| 체력 0 | 스테이지 재시작 |
| 데드락 감지 (폭탄 소진/돌 배치 불능으로 해법 소멸) | 재시작 안내 |
| 수동 재시작 | 항상 가능 (퍼즐 게임 필수 UX) |

## 10. 핵심 루프

```
룸 탐색 → 장애물 판정(나무/물/구덩이/문)
  → 필요 능력 식별 → 캐릭터 전환(무비용)
  → 돌 밀기 · 땅굴 · 수영 · 폭탄 순서 해결  [모든 이동 = 체력 소모]
  → 귤 수급으로 체력 유지 → 열쇠 확보 → 상자 개방
  → 클리어 (잔여 체력 = 점수) → [열기구 막간] → 다음 스테이지
```
